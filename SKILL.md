# 项目知识图谱技能

为任意项目生成结构化知识图谱（`PROJECT_MAP.md`），并提供检索工具按需查询，大幅节省 AI 编程时的 token 消耗。

## 何时调用

- **新项目初始化**：用户要求"分析项目结构""生成知识图谱""创建项目地图"时
- **动手前检索**：修改代码前需要了解项目架构、模块依赖、API 接口等信息时
- **变更后同步**：完成代码修改后，需要更新知识图谱对应章节时
- **跨项目快速上手**：接手陌生项目，需要快速建立全局认知时

## 核心组件

| 组件 | 位置 | 作用 |
|------|------|------|
| `PROJECT_MAP.md` | 目标项目根目录 | 知识图谱本体（AI 按项目实际情况编写） |
| `tools/search_map.py` | 本 skill 目录 | 检索工具（通用，适配任意项目） |
| `templates/PROJECT_MAP_TEMPLATE.md` | 本 skill 目录 | 空白章节模板（供 AI 参考） |
| `templates/AGENTS_PATCH.md` | 本 skill 目录 | AGENTS.md 补丁片段（引导检索+同步规则） |

## 工作流一：初始化知识图谱

当用户要求分析项目并生成知识图谱时，按以下步骤执行：

### 步骤 1：深度分析项目

使用 Task 子代理并行探索项目的各个层面（根据项目规模调整并行数量）：

- **架构入口**：主入口文件、配置管理、启动流程、核心设计模式
- **核心模块**：业务逻辑层、数据层、服务层、中间件
- **外部接口**：API 定义、第三方集成、SDK 调用
- **数据模型**：ORM 模型、数据库结构、数据流
- **工具/基础设施**：日志、加密、路径、异步辅助、测试

每个子代理返回详细的分析报告。

### 步骤 2：生成 PROJECT_MAP.md

基于分析结果，在项目根目录生成 `PROJECT_MAP.md`。

**参考 `templates/PROJECT_MAP_TEMPLATE.md` 的章节结构，但所有内容必须基于实际代码分析填写，不要留空模板占位符。**

章节编号固定 0-10，第 8 章根据项目特点自定义（如 Agent 工具、插件体系、事件系统等）。

编写要求：
- 每个章节必须有实质内容，基于代码分析填写
- 第 10 章「已知风险与禁忌」尤其重要——列出具体的修改禁忌和已知风险
- 使用 Mermaid 图表展示架构和流程
- 文件路径使用可点击的 markdown 链接

### 步骤 3：部署检索工具

将 `tools/search_map.py` 复制到目标项目的 `scripts/` 目录：

```bash
cp tools/search_map.py <目标项目>/scripts/search_map.py
```

如果项目没有 `scripts/` 目录，创建它。

### 步骤 4：更新 AGENTS.md

参考 `templates/AGENTS_PATCH.md`，在目标项目的 `AGENTS.md` 开头添加：
- 检索引导章节（引导 AI 使用 `search_map.py` 按需查询）
- 知识图谱同步规则（要求代码变更后立即更新 `PROJECT_MAP.md`）

章节编号根据目标项目 AGENTS.md 现有结构调整。

## 工作流二：检索知识图谱

修改代码前，使用检索工具了解相关上下文：

```
python scripts/search_map.py list              # 先看有哪些章节
python scripts/search_map.py 6                 # 查看第6章「数据模型」
python scripts/search_map.py "外键"             # 搜索外键相关内容
python scripts/search_map.py --map docs/X.md list  # 指定其他图谱文件
```

### 检索规则

- **章节号查询**：返回该章节完整内容（含子章节）
- **关键词搜索**：返回匹配行 + 前后 2 行上下文，`>` 标记命中行
- **忽略空格匹配**：搜索 "DI容器" 能匹配文本中的 "DI 容器"
- 每章节最多显示 8 处匹配，避免输出过长

### 重要提醒

检索工具帮你定位信息位置，**动手修改前仍需 Read 实际代码确认**。知识图谱是"地图"，不是"地形"本身。

## 工作流三：变更后同步

完成代码修改后，检查需要同步的章节：

| 变更类型 | 需同步的章节 |
|---------|------------|
| 新增/删除文件 | 第2章 目录结构 |
| 修改架构/分层 | 第1章 架构层次、第3章 模块依赖 |
| 新增/修改 API | 第7章 API 接口定义 |
| 修改数据模型 | 第6章 数据模型 |
| 新增/修改工具/插件 | 第8章 项目特有章节 |
| 修复已知风险 | 第10章 已知风险与禁忌（移除已修复项） |
| 新增依赖 | 第9章 技术栈组件 |
| 修改核心流程 | 第4章 核心业务流程、第5章 关键函数调用链 |

同步后，在 `PROJECT_MAP.md` 末尾更新最后修改日期。

## 检索工具参数说明

```
search_map.py [query] [--map <路径>]

参数:
  query        章节号(如 3)、关键词(如 'DI容器')、或 list
  --map, -m    知识图谱文件路径，默认 PROJECT_MAP.md

示例:
  search_map.py list                    # 列出所有章节
  search_map.py 7.3                     # 查看第7.3节
  search_map.py "消息队列"               # 搜索关键词
  search_map.py list --map docs/X.md    # 指定图谱文件
```

无第三方依赖，仅需 Python 3.8+。
