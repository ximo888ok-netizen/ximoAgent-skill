## X. 项目结构检索（动手前必读）

**在修改任何代码前，先用检索工具了解项目结构，避免盲目猜测。**

项目知识图谱位于 `PROJECT_MAP.md`（10章，覆盖架构/目录/依赖/流程/数据模型/API/项目特有体系/技术栈/风险禁忌）。
**不要直接读取整个文件**（浪费 token），使用检索工具按需查询：

```
python scripts/search_map.py list              # 列出所有章节（先运行这个了解全貌）
python scripts/search_map.py 2                 # 查看第2章「目录结构」完整内容
python scripts/search_map.py 7.3               # 查看第7.3节
python scripts/search_map.py "关键词"           # 按关键词搜索相关章节
```

**典型工作流：**
1. `list` → 了解有哪些章节
2. 按章节号查看相关章节
3. 按关键词搜索交叉信息
4. **必读第10章**（已知风险与禁忌）→ 避免踩坑

检索工具返回的是相关片段而非全文，大幅节省 token。

## Y. 知识图谱同步

**任何项目变动完成后，立即更新 `PROJECT_MAP.md`。**

无论是新增/删除文件、修改架构、调整依赖、变更API、新增工具、修改数据模型，还是修复已知风险，都必须在完成代码改动后同步更新对应章节，保持知识图谱与代码一致。遗漏同步会导致后续 AI 基于过时信息做出错误判断。
