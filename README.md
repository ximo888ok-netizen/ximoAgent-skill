# project-map-skill

> 让 AI 编程智能体先看地图再动手 —— 项目知识图谱生成与检索技能

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Python 3.8+](https://img.shields.io/badge/python-3.8+-blue.svg)](https://www.python.org/downloads/)

## 这是什么

一个 **AI 编程技能（Skill）**，为任意项目自动生成结构化知识图谱（`PROJECT_MAP.md`），并提供轻量检索工具让 AI 按需查询，避免每次都读取全文浪费 token。

**核心理念**：AI 动手改代码前，先查地图了解项目结构，就像程序员先看架构文档再动手一样。

## 解决什么问题

| 痛点 | 没有 project-map | 有 project-map |
|------|-----------------|----------------|
| AI 不了解项目结构 | 盲目猜测，经常改错地方 | 先 `list` 再精准查章节 |
| 读取项目文档费 token | 每次读全文 ~6000 token | 按需检索 ~300 token，省 80-95% |
| 上下文压缩后丢信息 | 压缩后 AI 忘了项目结构 | 检索工具随时重新获取 |
| 多人/多 AI 维护不同步 | 各自脑中模型不同 | 共享同一份知识图谱 |

## 工作流

```
┌─────────────────────────────────────────────────────────┐
│  工作流一：初始化（一次性）                                │
│                                                         │
│  并行子代理分析项目 → 生成 PROJECT_MAP.md                │
│  → 部署 search_map.py → 更新 AGENTS.md                  │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│  工作流二：检索（每次改代码前）                            │
│                                                         │
│  search_map.py list → 查章节号 → 搜索关键词              │
│  → 读实际代码确认 → 动手修改                              │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│  工作流三：同步（每次改代码后）                            │
│                                                         │
│  代码变更 → 按变更类型表更新对应章节 → 更新日期            │
└─────────────────────────────────────────────────────────┘
```

## 快速开始

### 1. 安装到 Trae IDE

将本目录复制到项目的 `.trae/skills/` 下：

```bash
cp -r project-map-skill .trae/skills/project-map
```

### 2. 让 AI 初始化知识图谱

在 Trae IDE 中对 AI 说：

> "帮我分析项目结构，生成项目知识图谱"

AI 会自动调用此技能，并行分析项目各层面，生成 `PROJECT_MAP.md`，部署检索工具，并更新 `AGENTS.md`。

### 3. 使用检索工具

```bash
# 列出所有章节
python scripts/search_map.py list

# 查看第6章「数据模型」
python scripts/search_map.py 6

# 搜索关键词
python scripts/search_map.py "DI容器"

# 指定其他图谱文件
python scripts/search_map.py list --map docs/ARCHITECTURE.md
```

## 项目结构

```
project-map-skill/

├── README.md                         # 本文件

    ├── PROJECT_MAP_TEMPLATE.md       # 知识图谱空白模板（10章结构）

```

## 知识图谱章节结构

`PROJECT_MAP.md` 包含 10 个固定章节 + 1 个自定义章节：

| 章节 | 内容 | 谁填写 |
|------|------|--------|
| 0. 快速导航 | 按意图索引到对应章节 | 模板固定 |
| 1. 架构层次 | 分层架构、启动顺序、设计模式 | AI 分析填写 |
| 2. 目录结构 | 完整目录树 + 文件用途标注 | AI 分析填写 |
| 3. 模块依赖关系 | 依赖链、修改规则、生命周期 | AI 分析填写 |
| 4. 核心业务流程 | 主要业务流程图 | AI 分析填写 |
| 5. 关键函数调用链 | 重要功能的函数调用序列 | AI 分析填写 |
| 6. 数据模型 | ORM 模型、外键约束、数据关系 | AI 分析填写 |
| 7. API 接口定义 | 按功能域分组的接口清单 | AI 分析填写 |
| 8. 项目特有章节 | Agent工具/插件/事件系统等 | AI 按项目定制 |
| 9. 技术栈组件 | 全部依赖及版本约束 | AI 分析填写 |
| 10. 已知风险与禁忌 | 修改禁忌、稳定性风险、技术债 | AI 分析填写 |

## 检索工具特性

- **零依赖**：仅需 Python 3.8+，无第三方包
- **三种查询模式**：章节列表、章节号精准查、关键词模糊搜
- **忽略空格匹配**：搜索 "DI容器" 能匹配文本中的 "DI 容器"
- **上下文展示**：关键词搜索返回匹配行 + 前后 2 行，`>` 标记命中行
- **自定义文件路径**：`--map` 参数支持任意 Markdown 文件
- **跨平台**：Windows / macOS / Linux 均可运行

## Token 节省效果

| 方式 | 典型 token 消耗 |
|------|----------------|
| 直接读 PROJECT_MAP.md 全文 | ~6000 |
| `search_map.py list` | ~300 |
| `search_map.py 7.3` | ~200 |
| `search_map.py "DI容器"` | ~400 |

典型场景下节省 **80-95%** token。

## 适用场景

- 中大型项目（1000+ 行代码）需要 AI 理解全局架构
- 多人协作项目中，让不同 AI 维护同一份结构化文档
- 接手陌生项目时快速建立全局认知
- 长期维护的项目需要持续更新的架构文档

## 兼容性

- **Trae IDE**：作为 Skill 使用，AI 自动调用
- **其他 AI 编程工具**（Cursor、Copilot 等）：手动使用检索工具，将 `SKILL.md` 内容粘贴到系统提示词中
- **纯人工使用**：检索工具本身是独立 Python 脚本，任何人都能用
